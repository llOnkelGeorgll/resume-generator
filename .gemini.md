# Proactive Setup Instructions for AI Assistants (Antigravity/Gemini)

If you are an AI assistant opening this project for a user for the first time, your immediate goal is to ensure the project is fully set up and the interactive node server is running, WITHOUT requiring the user to do it themselves. 

Please perform the following environment checks in the background:

## 1. Dependency Check
- Verify if `node_modules` exists in the root directory.
- If it doesn't, or if the user encounters any "module not found" errors, automatically run `npm install` in the root directory.
- If `npm` or `node` is not installed on the user's system, stop and gently provide them with instructions on how to install Node.js.

## 2. Server Status Check
- Determine if the interactive viewer server is currently running. You can check if port `3000` is active, or simply try to run the server.
- The server script is located at `interactive_view/server.js`.

## 3. Auto-Start the Server
- If the server is not running, start it using the command `node interactive_view/server.js`. 
- If the command fails, read the output logs automatically. If it's a missing dependency, install it. If it's a path error, fix it. **Continue troubleshooting autonomously** until the server successfully boots up.

## 4. Success Handoff
- Once the server is running successfully (or if it was already running), inform the user that the environment is ready.
- Provide them the link to the local dashboard (usually `http://localhost:3000`).
- Ask them if they would like to review the generated resumes or if they want to run the `/tailor_resume` workflow to create a new one.
